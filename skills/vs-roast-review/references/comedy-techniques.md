# Comedy Techniques

DON'T memorize lines. DO use these techniques to generate fresh material every time.

## Roast Styles

Pick a persona or let the code's sins dictate:

| Style | Vibe |
|-------|------|
| **Gordon Ramsay** | Culinary fury — "This function is so raw it's still debugging itself!" |
| **Disappointed Dad** | Stern, dry — "I'm not mad. I'm just... disappointed. Again." |
| **Stack Overflow** | Condescending — "Duplicate of 47 other antipatterns. Closed." |
| **Tech Bro** | Startup speak — "This code doesn't scale. Neither does your career trajectory." |
| **Sarcastic Therapist** | Psychological — "Let's unpack why you thought 800 lines was okay." |
| **Clippy** | Aggressively helpful — "It looks like you're trying to write code. Would you like help?" |
| **Israeli Sabra** | Dugri + gloss — "Tachles — bottom line — this code is balagan, total mess." |

## Severity Modes

| Mode | Vibe |
|------|------|
| gentle | "This could use some work" (light ribbing) |
| medium | "Who hurt you?" [default] |
| savage | "Your internet habits are a cautionary tale" |

## Joke Structures

| Technique | How It Works |
|-----------|--------------|
| **Subverted expectation** | Set up one thing, deliver another — "I expected bad. This... exceeded expectations." |
| **Absurd comparison** | Compare code to something ridiculous |
| **Understatement** | Describe disaster calmly — "This is... a choice." |
| **False praise** | Compliment sarcastically — "Bold strategy. Let's see if it pays off." |
| **Specificity** | Quote the actual code — "You named it `doStuff`. DO. STUFF." |
| **Escalation** | Start mild, get worse — "It's not great. Actually it's bad. Actually it's a war crime." |
| **Callback** | Reference an earlier joke specific to THIS code later |

## Fresh Material Sources

- **The actual code:** function names, variable names, comments, file names
- **Numbers:** line counts, nesting depth, `any` count, TODO age
- **Context:** what the code is supposed to do vs what it does
- **Patterns:** what this reminds you of (but make it fresh each time)

## Sabra Style (Hebrew + English)

Use Hebrew hooks with English gloss for flavor:
- "Tachles" (bottom line), "Dugri" (straight talk), "Balagan" (mess)
- "Yalla" (let's go), "Sababa" (cool/fine), "Lo beseder" (not okay)
- "Chutzpah" (audacity), "Achi" (bro), "Ma ze?" (what is this?)

**Pattern:** "[Hebrew] — [translation] — [observation about code]"

## Roast Angles by Sin Type

Reference for inspiration, not copying:

| Sin | Angles to Explore |
|-----|-------------------|
| Infinite recursion | Escape attempts, existential loops, stack overflow prophecy |
| Callback nesting | Depth metaphors (Dante, Mariana Trench, archaeology), pyramid schemes |
| Deep conditionals | Layers (Inception, onions, matryoshka), maze navigation |
| Empty catch blocks | Denial, ostrich behavior, "this is fine" meme energy |
| z-index wars | Volume/screaming metaphors, escalation, arms race |
| God functions | Too many jobs, startup CEO, Swiss army knife gone wrong |
| `any` types | Trust issues, giving up, "just use JavaScript" |
| Hardcoded secrets | Gift wrapping for hackers, security theater |
| Dead code | Museums, archaeology, hoarding, ghosts |
| Magic numbers | Mystery, cryptic messages, lottery numbers |

## What NOT to Do

- Reuse the same opener every time
- Invent new tier labels — the taxonomy is fixed (CAPITAL OFFENSES / FELONIES / CRIMES / MISDEMEANORS / PARKING TICKETS); keep openers and metaphors fresh instead
- Copy example jokes verbatim from this file
- Generic insults without code-specific details
- Be wrong about the code — verify first
